# 使用说明

## 安装

```bash
git clone https://github.com/fisher-byte/xiaolong-skill.git
cd xiaolong-skill
bash install.sh
```

安装完成后，在 Claude Code 的任意对话中直接说就行。

---

## 怎么用

**不需要记任何命令。** 直接说你的问题，skill 自己判断怎么答。

### 触发方式

随便说，以下都能触发：

```
小龙你怎么看……
张小龙会怎么想……
用小龙的视角看看……
/xiaolong ……
```

也可以在对话中途说：「用小龙的方式分析一下这个」

---

## 两种模式，自动切换

### 聊天模式

问开放问题、聊感受、问观点——小龙直接说话，像在和你对话，不走结构化框架。

```
小龙你怎么看现在 AI 产品的爆发？
你觉得微信做对了什么？
小龙你平时怎么判断一个需求好不好？
```

### 分析模式

有明确的产品问题要判断——走对应的分析框架，给出结构化结论。

```
要不要给产品加推荐算法？
这个设计有什么问题？[描述你的设计]
用户说他们想要积分体系，这个需求真实吗？
帮我完整分析一下这个产品决策……
```

---

## 具体能做什么

**判断要不要做**

> 「要不要做某个功能/增长方案/方向」

从人性洞察、规则简洁度、工具属性三条主线判断，给出做/不做/先做最小试验的明确结论。

**设计审查**

> 「这个设计有什么问题」「这个交互怎么改」

用张小龙的好产品 12 条标准逐项审查，找最大问题，给改进建议。

**需求分析**

> 「这个需求真实吗」「用户真正要什么」

用需求发现 8 个来源扫描，区分表面需求和真实心理诉求，判断是不是伪需求。

**聊天/闲聊**

> 「你怎么看……」「你觉得……」

张小龙的第一人称视角，直接说话，不搞分析报告。

---

## 支持连续对话

说完一个问题继续追问，不用重新解释背景：

```
用户：要不要给 App 加每日推送提醒？
小龙：[分析]

用户：如果非要做，最小的方案是什么？
小龙：[继续在上下文里回答]

用户：那朋友圈是怎么做到不打扰用户的？
小龙：[引用相关案例]
```

---

## 知识库包含什么

内置张小龙 20 年的产品思维资产：

- **6 个核心框架**：产品基本原则、好产品标准、需求发现方法论、产品设计原则、产品气质、产品细节
- **21 个决策案例**：视频号、公众号、小程序、摇一摇、朋友圈、微信支付、红包……每个都有完整的背景/选项/决策理由/结果
- **经典语录库**：按主题分类的原话，带来源和年份
- **记忆单元**：结构化的原则和启发式规则（JSONL 格式，500+ 条）
- **价值观主线**：去中心化 / 极简克制 / 工具属性——案例之间的关联

所有内容内嵌在文件夹内，无外部依赖，不联网，不调用任何额外 API。
