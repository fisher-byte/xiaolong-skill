# 小龙你怎么看

> 用张小龙的产品哲学，分析你的产品问题。

一个 Claude Code Skill。把张小龙 20 年的产品思维——框架、案例、原话、价值观——装进一个可以直接对话的助手。

直接说你的问题，不需要选命令，不需要填表。

---

## 安装

**方式一：ZIP 安装（推荐）**

下载 [最新 Release](https://github.com/fisher-byte/xiaolong-skill/releases)，在 Claude Code 的 Skills 管理界面上传 ZIP 文件。

**方式二：Git 安装**

```bash
git clone https://github.com/fisher-byte/xiaolong-skill.git
cd xiaolong-skill
bash install.sh
```

安装完成后，在 Claude Code 任意对话中直接触发：

```
小龙你怎么看，要不要给产品加推荐算法？
```

```
/xiaolong 这个设计有什么问题？
```

```
张小龙会怎么看「用户说想要积分体系」这个需求？
```

详细说明见 [USAGE.md](./USAGE.md)

## 两种模式，自动切换

不需要选模式，skill 根据你说的内容自动判断：

### 聊天模式

问开放问题、聊感受、聊宏观——小龙直接说话，像在和你对话。

```
小龙你怎么看现在 AI 产品的爆发？
你觉得微信做对了什么？
```

> 我有一个问题。这些 AI 产品，用户用完之后，感觉是什么？
>
> 大多数人用完之后的感觉是——哦，挺厉害的。然后就没有了。
>
> 这个感觉和当年用摇一摇是不一样的。摇一摇用完之后，用户想的是：还有谁在摇？我再试一次。
>
> 所以问题不是 AI 够不够强。问题是：你有没有找到一个简单的规则，让用户产生那种想再来一次的感觉。

### 分析模式

有明确的产品决策要做——走对应框架，给结构化结论。

| 你说的是 | 走的框架 |
|---------|--------|
| 要不要做 X | 人性 × 规则 × 工具属性，三条主线 |
| 这个设计有什么问题 | 好产品 12 条标准，找最大问题 |
| 这个需求真实吗 | 需求发现 8 个来源，找真实心理诉求 |
| 给我完整分析 | 三维决策框架 + 证据清单 |

## 知识库

内置张小龙 20 年的产品思维资产，所有内容离线可用：

```
data/
├── frameworks/        # 6 个核心框架（人×规则×环境、好产品标准、需求发现…）
├── decision-cases/    # 21 个决策案例（视频号、小程序、摇一摇、朋友圈…）
├── quotes/            # 经典原话库（按主题分类，带来源年份）
├── memory/            # 结构化记忆单元 500+ 条（JSONL）
├── evolution-timeline.md  # 产品观念演化时间线 2000–2021
└── profile.md         # 人物背景
```

案例之间通过三条价值观主线关联——**去中心化 / 极简克制 / 工具属性**——引用某个决策时能说清它背后的更大逻辑，而不只是孤立地搬一句话。

## 语气

模拟张小龙的说话方式：从问题或现象开始，慢慢建构，最后才点出原则。有停顿感，有反问，不废话。

✅ **像这样**：
> "规则太复杂了。一句话都说不清楚。"
> "用户要的不是这个功能，他要的是那种感觉。"
> "放大到一亿用户，这会变成什么？"

❌ **不像这样**：
> "从用户体验角度综合分析，建议您考虑以下几个优化路径……"
