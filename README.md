# 小龙你怎么看

> 用张小龙的产品哲学，分析你的产品问题。

一个 Claude Code Skill，把 20 年的张小龙产品思维浓缩成一个可以直接对话的助手。

---

## 安装

```bash
git clone https://github.com/fisher-byte/xiaolong-skill.git
cd xiaolong-skill
bash install.sh
```

安装完成后，在 Claude Code 的任意对话中直接用：

```
小龙你怎么看，我们要不要给 App 加推荐算法？
```

```
/xiaolong 这个设计有什么问题？[描述你的设计]
```

```
张小龙会怎么看「用户说他们想要积分体系」这个需求？
```

详细使用说明见 [USAGE.md](./USAGE.md)

---

## 它能做什么

**一个入口，对话式，自动识别你的问题类型：**

| 你问的是… | 小龙怎么答 |
|---------|----------|
| 要不要做某个功能/方向 | 人性洞察 × 极简规则 × 工具属性，三条主线判断 |
| 这个设计好不好 | 好产品 12 条标准逐项审查，找最大问题 |
| 这个需求真实吗 | 8 个需求来源扫描，找真实心理诉求 |
| 给我完整分析 | 人 × 规则 × 环境三维框架 + 证据引用 |

不需要选框架，不需要填表，直接说问题。

## 知识库

内置张小龙 20 年的产品思维资产：

```
data/
├── frameworks/          # 核心框架（人×规则×环境、好产品标准、需求发现）
├── decision-cases/      # 21 个经典决策案例（视频号、小程序、朋友圈…）
├── quotes/              # 经典原话库
├── memory/              # 结构化记忆单元（JSONL）
├── evolution-timeline.md  # 产品观念演化时间线（2000-2021）
└── profile.md           # 人物基本信息
```

所有知识内嵌在 skill 文件夹内，无外部依赖，不联网，不调用任何 API。

## 风格说明

回答风格模拟张小龙：**直接、有反问、不废话**

> "规则太复杂了。一句话都说不清楚。"
> "用户要的不是功能，他要的是那种感觉。"
> "放大到一亿用户，这会变成什么？"

不是咨询报告腔，不是 bullet point 堆砌，就是一个产品人在直接跟你说话。
