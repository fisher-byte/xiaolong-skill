# Case-08: "看一看"与"搜一搜"的克制设计决策

## 元信息
- **决策者**: 张小龙、微信团队
- **决策时间**: 2017年11月（微信6.5.19版本）
- **决策类型**: 内容消费入口设计
- **证据等级**: A级（张小龙2020公开课演讲）
- **案例状态**: 核心案例

---

## 1. 决策背景

### 微信的内容困境（2016-2017）

**问题1: 内容分发效率低**
- **订阅号打开率下降**: 从>10%（2014）跌至<5%（2017）
- **信息过载**: 用户订阅数膨胀，无法有效消费
- **优质内容埋没**: 好内容被淹没在订阅列表中

**问题2: 搜索能力不足**
- **微信搜索**: 仅支持好友/聊天记录搜索（无内容搜索）
- **用户需求**: 希望搜索公众号文章、朋友圈内容
- **竞争压力**: 百度、搜狗等竞争对手虎视眈眈

### 行业解决方案对比

| 产品 | 内容分发 | 搜索 | 特点 |
|-----|---------|------|------|
| **今日头条** | 算法推荐信息流 | 通用搜索引擎 | 最大化消费时长 |
| **知乎** | Feed流 + 推荐 | 专业内容搜索 | 知识社区 |
| **微博** | 信息流 + 热搜 | 实时搜索 | 热点传播 |
| **微信** | ？ | ？ | 克制产品哲学 |

---

## 2. 决策选项

### 选项A: 激进改版（学习今日头条）
**做法**:
- 订阅号改为算法推荐信息流
- 首页增加推荐Feed
- 搜索入口前置（底部Tab）

**优势**:
- 快速提升内容消费
- 增加用户停留时长
- 符合行业趋势

**劣势**:
- 违背"用完即走"理念
- 用户反感算法干预
- 失去微信特色

### 选项B: 保守策略（不做）
**做法**:
- 保持现状
- 专注通讯工具定位
- 拒绝内容化

**优势**:
- 保持产品克制
- 避免争议
- 专注核心价值

**劣势**:
- 用户需求未满足
- 创作者流失
- 被竞品分流

### 选项C: 克制设计（独立入口 + 社交推荐）✅
**做法**:
- 独立入口："看一看"、"搜一搜"
- 不改变主流程（聊天仍是核心）
- 社交推荐为主（算法为辅）

---

## 3. 决策逻辑

### 张小龙的产品哲学

**核心原则**:
> "我们做了'看一看'和'搜一搜'，但我们不希望它们成为主入口。微信是一个工具，不是内容平台。"

**设计约束**:
1. **不改变主流程**: 聊天仍是首页，内容入口藏在"发现"页
2. **社交推荐优先**: "好友在看"而非"为你推荐"
3. **用户可控**: 可以隐藏"看一看"、"搜一搜"入口
4. **克制推送**: 不主动推送内容（用户主动点击才看到）

### "看一看"设计

**入口位置**:
- 藏在"发现"页（不是底部Tab）
- 与朋友圈、小程序并列（非优先级）

**内容来源**:
1. **好友在看**（主要）: 基于社交推荐
2. **精选内容**（辅助）: 算法辅助排序
3. **热点话题**（补充）: 编辑精选

**张小龙2020年解释**:
> "我们在看一看里实验了社交推荐，看起来效果还不错。它是一种通过好友之间的互相推荐来扩大人的选择范围。"

### "搜一搜"设计

**入口位置**:
- 主搜索框（顶部）点击后展开
- 多Tab设计：全部/朋友圈/公众号/小程序/音乐/表情

**搜索范围**:
- 微信生态内容（不做通用搜索引擎）
- 公众号文章、朋友圈、小程序等

**克制点**:
- 不做SEO排名（避免商业化干预）
- 不做搜索广告（保持纯净）
- 不外链百度等（保持生态封闭）

## 4. 执行证据

### 阶段1: 上线初期（2017.11-2018）

**微信6.5.19版本**:
- "看一看"、"搜一搜"同时上线
- 入口在"发现"页底部

**用户反应**:
- 低调上线，未引发争议（与订阅号改版形成对比）
- 使用率低（多数用户不知道）
- 创作者关注度低（流量贡献<5%）

### 阶段2: 逐步优化（2018-2020）

**"看一看"演进**:
- 2018: 增加"精选"tab（算法推荐）
- 2019: 引入AI推荐
- 2020: 增加"直播"tab

**"搜一搜"演进**:
- 2018: 增加品牌官方区
- 2019: 支持搜索视频号
- 2020: 增加"朋友搜过"（社交推荐）

### 阶段3: 战略地位提升（2020+）

**张小龙2020年反思**:
> "看一看和搜一搜是微信的两个重要入口，但我们不希望用户在这里消耗太多时间。"

**数据表现**（未公开具体数据，行业估计）:
- 看一看DAU: ~1-2亿（10-20%微信用户）
- 搜一搜DAU: ~5000万-1亿
- 仍远低于主流内容平台（今日头条DAU >4亿）

---

## 5. 决策结果

### 短期影响（2017-2019）
- **守住阵地**: 避免被今日头条完全分流
- **用户接受**: 低调上线未引发反感
- **流量贡献**: 为公众号带来10-15%增量流量

### 长期影响（2019-2026）
- **战略价值**: 成为微信内容生态的重要组成部分
- **克制成功**: 始终未成为主入口（保持微信工具定位）
- **社交推荐验证**: 证明"社交推荐"可行（不必完全依赖算法）

### 与竞品对比

| | 微信（看一看/搜一搜） | 今日头条 |
|---|---|---|
| **入口位置** | "发现"页（非主入口） | 首页Tab（主入口） |
| **推荐逻辑** | 社交推荐为主 | 算法推荐 |
| **DAU** | 1-2亿（估计） | 4亿+ |
| **用户时长** | <10分钟/日 | 80+分钟/日 |
| **产品定位** | 辅助功能 | 核心功能 |

**结论**: 微信成功守住阵地，但未能真正挑战今日头条

## 6. 决策启发式

### 启发式1: 独立入口 vs 改造主流程

**适用场景**: 增加新功能但可能引发争议  
**决策规则**:
```
IF 新功能与产品核心定位有冲突
THEN 做独立入口（而非改造主流程）：
  - 保护核心体验（聊天仍是主流程）
  - 给用户选择权（可以不用新功能）
  - 降低争议（不强制改变用户习惯）
```

**微信的执行**:
- ❌ 不把首页改成信息流（保护聊天核心）
- ✅ 独立做"看一看"（藏在"发现"页）
- ✅ 用户可以隐藏入口（尊重选择）

**对比今日头条**:
- 今日头条：首页就是推荐信息流（核心功能）
- 微信：首页是聊天，"看一看"是辅助功能

### 启发式2: 社交推荐 > 算法推荐（符合产品定位）

**适用场景**: 内容分发机制选择  
**决策规则**:
```
IF 产品定位是"连接人"（而非内容消费）
THEN 优先社交推荐：
  - "好友在看"（基于社交关系）
  - "朋友搜过"（基于社交推荐）
  - 而非"为你推荐"（纯算法）
```

**微信的逻辑**:
- 微信是社交工具 → 推荐也应该基于社交
- 今日头条是内容平台 → 算法推荐最优

**张小龙的判断**:
> "社交推荐是一种通过好友之间的互相推荐来扩大人的选择范围。"

### 启发式3: 克制是长期主义

**适用场景**: 面对增长压力和竞争压力  
**决策规则**:
```
IF 可以通过激进功能快速增长
BUT 会损害长期用户体验
THEN 选择克制：
  - 不做主入口（避免喧宾夺主）
  - 不做推送（避免打扰用户）
  - 不做广告（保持纯净体验）
```

**微信的克制**:
- 看一看/搜一搜藏在"发现"页（而非底部Tab）
- 不主动推送内容（用户主动点击）
- 不做SEO排名（避免商业化干预）

**短期代价**:
- DAU低于竞品（1-2亿 vs 今日头条4亿+）
- 用户时长低（<10分钟 vs 80+分钟）

**长期收益**:
- 保持用户信任（工具定位未被破坏）
- 避免用户疲劳（不会因内容过载而卸载）
- 微信仍是国民级超级App

## 7. 与其他案例对比

### vs Case-06 (订阅号信息流改版)
- **订阅号**: 改造存量功能（引发争议）
- **看一看**: 新增独立入口（低调上线）
- **启示**: 新增比改造更容易被接受

### vs Case-01 (视频号战略)
- **视频号**: 独立Tab（战略级重视）
- **看一看**: "发现"页（辅助功能）
- **差异**: 战略地位决定入口权重

### vs 今日头条
- **今日头条**: 激进策略（首页推荐，最大化消费）
- **微信**: 克制策略（独立入口，保持工具定位）
- **结果**: 今日头条DAU更高，但微信保持长期信任
