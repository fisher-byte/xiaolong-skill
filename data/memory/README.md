# 张小龙记忆单元（Memory Units）

这套文件用于把“张小龙公开演讲/访谈/案例”的内容整理成可检索、可引用、可更新的记忆单元，供 `SKILL.md` 在回答时调用。

## 当前分工

- `memory_units.jsonl`：主库。面向默认回答使用，优先收录能直接支撑产品判断的高置信核心记忆。
- `memory_units.repo_candidates.jsonl`：候选库。用于补充检索面、发现遗漏主题、辅助扩充主库；不应成为默认正面判断的主要支撑。

## 使用原则

1. 前台像人，后台有证据  
   默认回答优先基于主库形成自然判断，不要把 low confidence、候选碎片、检索痕迹直接端到前台。

2. 主库优先，候选库后置  
   能用主库支撑的判断，不要先去引用候选库。只有主库缺口明显时，才用候选库补齐检索面。

3. 原话优先于二次概括  
   优先收录明确原话、明确案例、明确年份。候选库里那种同义碎片，更适合作为线索，不适合直接构成主回答。

4. 主库要像判断工具，不是摘抄仓库  
   每条主库记忆都应能直接支持一个产品判断。如果一条内容只能重复口号，不能帮助做取舍，就不该优先进入主库。

## 质量门槛

优先进入 `memory_units.jsonl` 的条目，建议满足下面至少 4 条：

1. 有明确年份或时期
2. 有明确出处文件
3. `evidence_quote` 接近原话，而不是过度转述
4. 能直接支撑一个决策判断
5. 有具体案例落地，而不是只有抽象口号
6. 与已有主库条目不重复

## 候选库的作用

候选库不是错，而是更像搜索索引：

- 一段原文被切成很多碎片
- 同义重复太多
- 只有方向，没有完整语境
- 更像补线索，不像直接可用的判断单元

所以候选库的正确用途是：发现线索、辅助扩库、补充检索面。
